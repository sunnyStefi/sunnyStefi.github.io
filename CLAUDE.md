## Overview

Personal blog/portfolio site (`sunnyStefi.github.io`) built with Jekyll using the `minima` theme (dark skin). Deployed to GitHub Pages via GitHub Actions on every push to `main`.

## Commands

```bash
bundle install              # Install Ruby gem dependencies
bundle exec jekyll serve    # Local dev server with live reload at http://localhost:4000
bundle exec jekyll build    # Production build into _site/ (what CI runs with JEKYLL_ENV=production)
bundle exec jekyll serve --drafts   # Include _drafts/ posts in the preview
```

There is no test suite or linter. Deployment is fully automated by `.github/workflows/deploy.yml` (Ruby 3.3) — no manual deploy step.

## Architecture

Content is driven entirely by Jekyll front matter and conventions; there is no application code.

- **Theme is gem-based (`minima`)**, so most layouts/styles live inside the gem, not this repo. Only the deltas are overridden locally:
  - `_layouts/home.html` and `_layouts/category.html` — both extend the minima `base` layout (which is *not* in this repo). `post` layout also comes from the gem.
  - `_sass/minima/` — SCSS overrides for theme styling.
  - `_includes/header.html`, `footer.html`, `custom-head.html` — override minima partials. `custom-head.html` injects a script forcing all external links to open in a new tab.

- **Navigation is category-driven.** The nav bar in `header.html` is generated by scanning `site.pages` for any page with `layout: category`, sorted by an `order:` front-matter field. To add a top-level nav section, create a markdown page like `engineering.md` with `layout: category`, a `category:`, a `permalink:`, and an `order:`.

- **Posts** live in `_posts/` named `YYYY-MM-DD-title.md`. URLs follow the permalink pattern `/:year/:month/:day/:title/` (set in `_config.yml`). A post's `category:` front matter assigns it to a category page; `category.html` filters posts via `where_exp` on `p.categories contains page.category`.

- **Post front matter** used by the listing templates (`home.html` / `category.html`):
  ```yaml
  ---
  layout: post
  title: "..."
  description: "..."        # rendered as the post excerpt in listings
  category: Agentic Engineering   # must match a category page's `category:`
  image: /assets/images/foo.jpg   # optional thumbnail in listings
  ---
  ```

- **Drafts** go in `_drafts/` (no date prefix needed) and only appear with `--drafts`.

## Key files

- `_config.yml` — site title, permalink scheme, theme skin, plugins (`jekyll-feed`, `jekyll-seo-tag`).
- `_drafts/`, `_posts/` — content. `_site/` and `.jekyll-cache/` are generated; do not edit.
